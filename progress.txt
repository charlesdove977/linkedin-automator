#### Architecture Decisions

1. **Markdown-based commands** - Each command is a .md file with structured behavior instructions for Claude to follow
2. **Reference docs** - Commands reference shared docs for consistent behavior (filtering, patterns, etc.)
3. **User templates** - Setup wizard copies templates to user's folder, populated with their config
4. **Chrome-first** - All browser automation uses Claude-in-Chrome MCP tools
5. **Human-in-the-loop** - Every message shown for approval before sending

#### Ready for Testing

Plugin structure complete. To test:
1. Install Claude-in-Chrome extension
2. Launch `claude --chrome`
3. Run `/linkedinoutreach` to configure
4. Try `/dmconnections` to send messages
